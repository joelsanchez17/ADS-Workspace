RISC-V Processor Debugging Environment

This platform helps you debug a multi-stage RISC-V processor written in Chisel. Upload your src folder, let the system detect your current architecture level, compile it with the matching system testbench, and inspect the processor cycle by cycle.

First time here? Start by uploading only your project src folder.
📄 RISC-V ISA Reference 📘 Chisel Documentation
Step 01

Upload your source folder

Select the folder containing your Chisel modules. No ZIP file and no local toolchain are required.

Step 02

Automatic level detection

The backend detects whether your design is basic, forwarding-enabled, memory-ready, or control-flow capable.

Step 03

Compile and simulate

Your uploaded design is compiled with the appropriate BinaryFile and live Chisel testbench.

Step 04

Debug visually

Use the editor, pipeline diagram, register bank, console logs, and Surfer waveforms to find bugs.

Before uploading
  • Upload the project src folder, not the full repository.
  • Keep hardware modules inside src/main/scala.
  • Student test files are ignored for safety and consistency.
  • You can use the system testbench or your own BinaryFile after upload.
